1、需要JAVA8或以上环境。
2、需要拥有磁盘根路径/lichkin-logs目录的读写权限。（日志文件将在此目录中读写）
3、需要拥有磁盘根路径/opt/security/connector.properties文件的读权限。（正式环境必须有此文件且需要配置，测试环境可以没有此文件。）
4、以上准备就绪后，可直接运行如下命令调用本应用：
java -jar lichkin-framework-cloud-connector.jar --T=thermometer --M=LichKin --I=1234567890 --DATAS=A01234567890
5、命令解释：
	5.1、java -jar cloud-connector.jar JAVA调用JAR包基础命令。
	5.2、后面是基于JAVA调用JAR包基础命令传递参数的方式（每个参数使用空格分开）基础上自定义的参数，参数顺序不敏感。
	5.3、自定义参数：使用【--】作为前缀，【=】作为键值对的分隔符。即【--】后，【=】前为自定义参数键，【=】后为自定义参数值。
	5.4、T：设备类型。如--T=thermometer，表示T（设置类型）的值为thermometer（温度计）。
	5.5、M：设备厂商。如--M=LichKin，表示M（设置厂商）的值为LichKin（某厂商）。
	5.6、I：设置ID。如--I=1234567890，表示I（设置ID）的值为1234567890（具体设备的唯一标识）。
	5.7、DATAS：设置发送的数据值。如--DATAS=A01234567890，表示DATAS（设置发送的数据值）的值为A01234567890（具体数据值）。
	5.8、T、M、I的组合值必须保证能够唯一的识别某设备，这将用于判断如何解析数据，以及推送的数据将发往哪个设备。
